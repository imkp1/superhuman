01 / 05 — the premise
The problem & the bet

AI-generated PRs
are a flood.
Most don't merge.

Maintainers are drowning in low-quality bot submissions. The contributors who succeed are the ones who read the repo's actual conventions, run its actual CI, and respond to feedback — before asking for review.

The bet — agent wins iff it

  1. Reads the repo's actual conventions before writing a line
  2. Runs the repo's actual CI before pushing a commit
  3. Scores itself against a calibrated rubric before review
  4. Stops early on hopeless runs — guards maintainer goodwill
Not a chatbot that writes code. A pipeline that ships PRs.  One thin orchestrator, six specialists, one scorer, and a calibration corpus that grows with every run.

02 / 05 — strategy
Strategy

One orchestrator.
Six specialists.

The orchestrator does zero domain work. It sequences, locks, and enforces thresholds. Everything else is a specialist with its own context window, its own JSON contract, and its own swappable implementation.

Why this shape wins

Swappable

Upgrading the reviewer doesn't touch the orchestrator. Every boundary is a JSON file on disk.

Context hygiene

Each agent runs in its own context window. No agent drowns in everything. No one-prompt monster.

Auditable

Every handoff is inspectable: current_contribution.json, scores, outcomes, mistakes.

04 / 05 — rails
Safety & trust

Hard rails.
Soft rails.

Hard rails are kernel-enforced or abort-on-violation. Soft rails are calibrated thresholds — tuned against the real corpus of past merge outcomes, not against intuition.

Hard railsnon-negotiable

flock(2) mutex

One run per repo, enforced by the kernel — not by JSON bookkeeping.

Fork-only push

origin is the fork; upstream is the source — never written.

AI-policy check

Reads CONTRIBUTING.md; aborts if AI-generated code is forbidden, quoting the line.

Rate-limit guard

Refuses to start with < 200 GitHub API calls remaining.

Impact audit

Before any refactor: enumerate callers, classify contexts, block on surprise blast radius.

Suspicious halt

resolve-comments stops on prompt-injection in a review; human decides next step.

Allowlisted CI

Builder only runs commands parsed out of .github/workflows/*.yml. No ad-hoc shell.

Soft railscalibrated, tunable

95 × 2 ship threshold

Scorer must cross 95% on two consecutive runs — single spikes are not merge-ready.

< 50 after iter 5 abort

Low score after many rounds means the plan is wrong. Stop; let the human redirect.

Weakest-first review

Dispatcher always targets the lowest-scoring dimension — no wasted passes on strong areas.

Plateau detection

Scorer flags dimensions that won't improve; orchestrator stops burning cycles on them.

Corpus calibration

Every outcome appends to merge_outcomes.jsonl — thresholds tune against real merges.

Mistakes logbook

Every CI failure, impact block, or crash appends to mistakes.md — growing institutional memory.

05 / 05 — what ships
What ships

A merge-ready PR.
A row in the corpus.

One run leaves behind a real pull request the maintainer can merge — and a record that makes the next run smarter. The loop closes around the calibration data, not the model.

Mistakes logged: 1  builder:ci_gate — pytest failure fixed iter 2

Operating principles

  1. Dispatch; don't implement.
  2. One contribution at a time per repo.
  3. Two consecutive ≥ 95 runs, never one.
  4. Abort early on hopeless runs.
  5. Record every outcome — wins and losses.
  6. Surface, don't retry, on security halts.
  7. Fork-only push target, always.
